Описание (набросок самой идеи проекта):

main - позволяет управлять finder, louder, state-machine

Finder:
сообщаем, где ищем плагины → 
формируем список найденных плагинов

Louder:
плагины переводим в модули →
импортируем модули →
валидируем модули →
при некорректной работе модуля выгружаем его →
формируем коллекцию корректных импортированных модулей

State-machine:
коллекцию плагинов(модулей) загружаем в машину состояний →
управляем состоянием плагинов


state-machine выполняет:
- init - плагин перечитывает свою конфигурацию или иные прочие подготовительные действия
- start - запуск плагина по имени
- stop - остановка плагина по имени
- start all - запуск всех плагинов
- stop all - остановка всех плагинов
- restart - остановка и запуск плагина
- reset - остановка, инициализация и запуск плагина
- get info - информация по плагину
- get all info - информация по всем плагинам
- get state - состояние плагина
- get all state - состояние всех плагинов
- get all error - вывести все плагины с ошибкой
- ???add plugin - добавить новый плагин
- ???remove plugin - выгрузить плагин

State плагинов:
- CREATED - плагин создан но не инициализирован
- INIT - плагин инициализирован и готов к старту
- STARTED - плагин запущен
- STOPPED - плагин остановлен
- FAILED - плагин с ошибкой

???Промежуточные State???: 
помечать как transition (STATE_ONE - transition - STETE_TWO)
- RELOAD - переход в состояние INIT
- STARTUP - переход в состояние START
- STOPPING - переход в состояние STOP
